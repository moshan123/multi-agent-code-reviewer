# Multi-Agent 代码审查助手 - Vibe Coding 演示指南

## 🎬 演示视频录制脚本

### 开场 (30 秒)

**画面**: 项目 README 页面

**旁白**:
> "大家好！今天我要展示如何用 Vibe Coding 的方式，从零开始构建一个 Multi-Agent 代码审查系统。
> 这个项目使用 MCP 协议，让多个 AI Agent 协作完成代码审查任务。"

---

### 第一部分：项目架构介绍 (1 分钟)

**画面**: 切换到架构图

**旁白**:
> "让我介绍一下项目架构：
> - 用户通过自然语言下达指令
> - Orchestrator Agent 分析请求并分发任务
> - 三个专业 Agent 并行执行：Security 检查安全问题、Quality 检查代码质量、Documentation 检查文档
> - 最后汇总结果生成报告"

---

### 第二部分：Vibe Coding 展示 (3 分钟)

**画面**: Claude Code 终端

**旁白**:
> "现在，让我用自然语言指挥 AI 帮我完成一个功能扩展。
> 比如，我想添加一个新的 Performance Agent 来检查性能问题。"

**演示步骤**:
1. 打开终端，输入自然语言指令
2. 展示 AI 生成的代码
3. 运行测试验证

**示例指令**:
> "帮我创建一个新的 Performance Agent，检查代码中的性能问题，比如循环中的数据库查询、未缓存的重复计算等"

---

### 第三部分：实际审查演示 (2 分钟)

**画面**: Dashboard 界面

**旁白**:
> "现在让我审查一个真实的 GitHub PR。
> 输入仓库名称和 PR 编号，点击开始审查..."

**演示步骤**:
1. 输入一个真实 PR
2. 展示审查进度
3. 查看审查结果

---

### 第四部分：多 Agent 协作过程 (1 分钟)

**画面**: 终端输出/日志

**旁白**:
> "大家可以看到，三个 Agent 正在并行工作：
> - Security Agent 发现了潜在的安全问题
> - Quality Agent 指出了代码风格问题
> - Documentation Agent 检查了文档完整性
> 最后由 Orchestrator 汇总生成完整报告"

---

### 结尾 (30 秒)

**画面**: 项目 GitHub 页面

**旁白**:
> "这就是用 Vibe Coding 方式构建的 Multi-Agent 代码审查系统。
> 通过自然语言指挥 AI，我们可以快速构建复杂的应用。
> 项目已开源，欢迎 Star 和贡献！"

## 🎯 关键信息点

1. **Vibe Coding 理念**: 用自然语言描述想法，让 AI 写代码
2. **技术亮点**: MCP 协议、多 Agent 协作、自动化审查
3. **实用价值**: 真实可用的开发工具，提高代码质量
4. **开源精神**: 项目开源，欢迎贡献

---

## 💡 演示技巧

1. **节奏控制**: 每个部分严格控制时间，不要拖沓
2. **视觉冲击**: Dashboard 界面要美观，分数变化要明显
3. **互动感**: 假装在和 AI 对话，展示自然语言编程的魅力
4. **高潮设计**: 在展示审查结果时，可以夸张一点"看，AI 发现了这么多问题！"
