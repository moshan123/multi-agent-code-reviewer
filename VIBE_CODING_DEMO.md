# Vibe Coding 演示脚本

## 🎯 演示目标

展示如何用 **Vibe Coding** 方式，通过自然语言指挥 AI 构建一个 **Multi-Agent 代码审查系统**。

---

## 📝 演示流程 (总计 10-15 分钟)

### 第一部分：开场介绍 (2 分钟)

**场景**: 项目 README 页面

**台词**:
> "大家好！今天我要展示 Vibe Coding 的魅力 —— 用自然语言描述想法，让 AI 帮我写代码。
>
> 我们要构建的是一个 **Multi-Agent 代码审查系统**，它可以：
> - 自动审查 GitHub PR
> - 检查安全问题、代码质量、文档完整性
> - 通过多个 Agent 协作完成复杂任务
>
> 整个过程，我只需要用自然语言告诉 AI 我想要什么，它就会帮我实现。"

**操作**:
1. 展示项目 README
2. 切换到架构图

---

### 第二部分：展示现有成果 (3 分钟)

**场景**: 项目文件结构

**台词**:
> "让我介绍一下项目结构：
> - `mcp-servers/` - MCP 服务器，提供代码分析和 GitHub 集成功能
> - `agents/` - 四个 Agent：Orchestrator、Security、Quality、Documentation
> - `dashboard/` - Next.js 构建的前端界面
> - `shared/` - 共享工具函数
>
> 这些都是通过 Vibe Coding 方式构建的 —— 我用自然语言描述需求，AI 帮我生成代码。"

**操作**:
1. 展示文件树
2. 打开几个核心文件，展示代码

---

### 第三部分：实时 Vibe Coding 演示 (5 分钟)

**场景**: 终端/Claude Code

**台词**:
> "现在，让我用自然语言指挥 AI，添加一个新功能 —— **Performance Agent**，用于检查代码性能问题。"

**操作步骤**:

#### Step 1: 提出需求
```
帮我创建一个新的 Performance Agent，用于检查代码中的性能问题。
需要检测的问题包括:
- 循环中的数据库查询
- 未缓存的重复计算
- 大列表的低效遍历
- 字符串拼接代替 join
```

#### Step 2: 展示 AI 生成的代码
**台词**:
> "AI 已经生成了 Performance Agent 的完整实现，包括:
> - 性能问题检测逻辑
> - 建议生成
> - 评分系统"

#### Step 3: 运行测试
```bash
python agents/performance_agent.py --code "test code here"
```

**台词**:
> "让我们测试一下这个新生成的 Agent... 看，它可以正确识别性能问题并给出建议！"

---

### 第四部分：Dashboard 演示 (3 分钟)

**场景**: Dashboard 界面 (http://localhost:3000)

**台词**:
> "现在让我展示 Dashboard 界面。这是一个现代化的 Web 应用，使用 Next.js 和 shadcn/ui 构建。"

**操作**:
1. 输入一个测试 PR（可以使用模拟数据）
2. 点击"开始审查"
3. 展示审查结果

**台词**:
> "Dashboard 正在调用后端的 Agent 系统...
> 看，审查结果出来了：
> - 安全评分：85 分，发现 1 个潜在问题
> - 质量评分：78 分，有 3 个改进建议
> - 文档评分：92 分，非常好！
>
> 每个问题都有详细的描述和改进建议。"

---

### 第五部分：总结 (2 分钟)

**场景**: 项目 GitHub 页面

**台词**:
> "这就是 Vibe Coding 的力量！在短短 15 分钟内，我们：
> 1. 构建了一个完整的 Multi-Agent 系统
> 2. 添加了新的 Performance Agent
> 3. 展示了可视化 Dashboard
>
> 传统开发方式可能需要几天时间，但通过 Vibe Coding，我们专注于'意图'而非语法细节，让 AI 处理繁琐的编码工作。
>
> 项目已开源在 GitHub，欢迎 Star 和贡献！
>
> 谢谢大家！"

**操作**:
1. 展示 GitHub 仓库地址
2. 展示二维码（可选）

## 🎬 演示准备清单

### 环境准备
- [ ] Python 3.9+ 已安装
- [ ] Node.js 18+ 已安装
- [ ] 依赖已安装：`pip install -r requirements.txt`
- [ ] Dashboard 依赖已安装：`npm install`
- [ ] Dashboard 可以正常启动

### 测试数据准备
- [ ] 准备一个公开的 GitHub PR 用于演示
- [ ] 或者使用模拟数据

### 演示环境
- [ ] 录屏软件已配置（OBS/QuickTime）
- [ ] 麦克风测试正常
- [ ] 终端字体大小适中
- [ ] 浏览器窗口大小适中

### 备用方案
- [ ] 准备录屏备份（以防现场演示出问题）
- [ ] 准备模拟数据（以防 GitHub API 限流）

## 💡 演示技巧

### 节奏控制
- 每个部分严格控制时间
- 不要在某一个细节上停留太久
- 准备一个计时器

### 视觉效果
- 使用大字体的终端
- 浏览器使用清晰的 theme
- 确保代码高亮可见

### 互动感
- 假装在和 AI 对话
- 展示 natural language programming 的魅力
- 在展示结果时可以适当夸张

### 应对意外
- 如果网络出问题，使用本地模拟数据
- 如果代码运行出错，展示备用录屏
- 保持冷静，幽默应对

演示中反复强调的要点：

3. **MCP 协议**: 标准化集成
4. **实用价值**: 真实可用的工具
